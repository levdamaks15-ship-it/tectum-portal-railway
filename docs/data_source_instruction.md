# Инструкция: Загрузка данных рапорта АЦИ

В этом файле вы можете фиксировать правила и коррективы для загрузки данных из отчетов АЦИ.

## Основные правила маппинга линий
Как было зафиксировано, на платформе мы используем названия линий строго:
- **Линия 1**
- **Линия 2**

В исторических и текущих рапортах (например, `рапорт_АЦИ 10.06.26..xlsx`) они могут называться по-другому. При загрузке и обработке данных (скрипты `import_aci.py`, `import_aci_instruction.py`) действует следующее правило сопоставления:

* **Хоргос** -> `Линия 1`
* **Шанхай** -> `Линия 2`

> **Внимание:** В скриптах уже заложена проверка: если встречается слово "хоргос" (в любом регистре), смена привязывается к "Линия 1". Для "шанхай" - к "Линия 2".

## Данные из рапорта АЦИ
В данный момент за основу берется файл `рапорт_АЦИ 10.06.26..xlsx`. Из него скрипты извлекают:
- Дату и смену (День/Ночь)
- Линию (согласно правилу выше)
- 1 сорт (в штуках/листах)
- Брак (в штуках/листах)

## Приход сырья (RawMaterialReceipt)

Приход сырья — это **независимая операция**, не связанная с производственным рапортом смены.

**Ключевые правила:**

1. **Отправка прихода:** Сменный мастер может отправлять приходы сырья в течение дня несколько раз. Для этого используется блок "Приход сырья", в котором заполняются базовые параметры смены: Дата, Смена, Линия, Мастер.

2. **Отдельная таблица:** Данные прихода хранятся в отдельной таблице `RawMaterialReceipt` (связь один-ко-многим со `Shift`). Каждая запись прихода — это отдельная строка.

3. **Экспорт в Google Sheets:** Каждая запись прихода выгружается как отдельная строка в лист "Приход сырья" Google Таблицы. Данные не суммируются с предыдущими приходами и не перезаписывают их.

4. **Не влияет на сводную таблицу на портале:** Наличие записей прихода **НЕ делает смену видимой** в сводной таблице (`/api/report/summary`). Сводная таблица отображает только смены с производственной активностью: план, формовка (ЛФМ), продукция на складе, ЗО, простои.

5. **Связь со сменой:** Технически приход привязан к `Shift` через `shift_id`. Это нужно для получения контекста (дата, смена, линия), но не означает, что приход "относится" к сменному рапорту. Сменный рапорт заполняется в конце смены отдельно.

## Фильтрация "пустых" смен в сводной таблице

Функция `get_report_summary()` в `main.py` содержит фильтр, который исключает из результатов смены без производственных данных:

```python
if plan_sheets_check == 0 and lfm_sheets_check == 0 and warehouse_gp_check == 0
   and zo_batches_check == 0 and not shift.zo_submitted and not shift.downtimes:
    continue
```

Обратите внимание: **`shift.receipts` НЕ проверяется** в этом условии. Приход сырья не делает смену "непустой".
